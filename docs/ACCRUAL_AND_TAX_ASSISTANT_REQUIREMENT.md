# Accrual Method and Tax Assistant Requirement

## Confirmed Direction
The accounting layer must include accrual-method strategy and calculation support. It is not enough for the system to remain only a simple ledger or cash-style tracker.

## Required Accounting Direction
The system should evolve into a tax-assistant-capable fiduciary and trust accounting environment.

## Required Accounting Support
- accounting method awareness
- accrual method strategies
- accrual calculations
- recognition date handling
- due date handling
- paid date handling
- receivable / payable style tracking
- period-based summary logic
- cash vs accrual comparison capability

## Tax Assistant Requirement
The system is intended to assist for tax purposes and should eventually support all IRS tax forms and data structures necessary to operate and administer a trust.

## Product Meaning
This confirms that the accounting layer is not merely optional bookkeeping. It is a required operational component of the overall trustee-first and fiduciary-first wealth management operating system.

## Suggested Build Direction
1. Build accounting layer phase 1 with accrual-aware fields
2. Add chart of accounts and ledger classification
3. Add trust financial summary views
4. Add tax assistant structures and reporting preparation layers
5. Expand into IRS trust-form support and filing-preparation workflows
