# Trustee App Build Doctrine

## Locked Product Identity
Trustee Management & Property Mapping App / Trust Operating System

## Core Vision
A guided, Rocket Lawyer–style trust administration system that helps a trustee create, manage, document, and maintain a trust, while mapping property, linking accounts, organizing records, and generating a final trust product packet.

## Locked Recommendations
- Multi-step create trust wizard, not one-page trust creation
- Rocket Lawyer–style explanations on important fields
- Review-before-finalize workflow
- Final trust packet output
- Rules engine by trust type
- Private/Public implemented as both visibility layer and workflow mode
- AI as explainer + next-step recommender, not legal decider
- Accounting method as trust-level configuration
- Readiness/completion tracking
- Persistent records, versioning, and final lock
- Trust detail and property detail as operating views, not dead ends
- Recommended follow-up documents and next actions based on answers

## Trust Types
- Revocable
- Irrevocable
- 98 Trust (advanced custom mode)

12. UCC Tracking
13. Tax Module

## AI Layer Rules
The AI should:
- Explain what a field/term means
- Explain why the question is asked
- Explain how choices change the workflow
- Suggest likely next steps
- Detect missing structure
- Translate administrative/legal language into plain English

The AI should not:
- Make final legal conclusions
- Guarantee legal outcomes
- Present suggestions as mandatory unless the system is explicitly configured to require a step

## Private / Public Interpretation
### Layer A — Record Visibility
- Private
- Internal
- Public-facing

### Layer B — Workflow Mode
- Private Office Mode
- Public Filing / External Mode

## Create Trust Wizard Requirements
The wizard must cover:
1. Identity
2. Trust Type and Purpose
3. Parties and Roles
4. Powers and Administration
5. Funding / Property / Initial Corpus
6. Tax and Accounting Preferences
7. Review and Finalization
8. Final Packet Generation

## Final Product Standard
The final product should be surgical, precise, detailed, and lawful in presentation. The app must guide the user through the full process and produce a finished trust packet with recommended supporting documents and next-step actions.
